## 2. Contexte Légal & Réglementaire

### Droit Foncier & Situation Post-Conflit

**Cadre légal:**
- Code foncier congolais (héritage français, weak enforcement)
- Dual: urban (Kinshasa formal) + rural (customary dominant)

**Situation conflit Kivu (2023-2024):**
- East Kivu: occupation illégale de propriétés (guerre 24+ ans)
- Déplacement interne: 6+ millions IDPs
- Property disputes: propriétaire pré-conflit vs. occupant actuel
- Justice: courts dysfunctional (Goma = no functioning judiciary)

**Reconstruction needs:**
- Land titling for IDP return
- Dispute resolution (multi-stakeholder: IDP + occupant + community)
- UN tracking (IDMC reporting, UNOPS coordination)
- Trust in institutions (land = symbolic restitution)

### Succession & Héritage

- **Droit chrétien:** Majorité (90%)
- **Droit coutumier:** Dominant zones rurales + Kivu (ethnic systems complex)
- **Droit islamique:** Minorité (5%, Kinshasa + Katanga)
- **SafeLandFridda:** Multi-system critical (Christian + customary Congolese + Islamic)

### Cadre Crypto & Blockchain

- **Politique:** Aucune régulation
- **UN backing:** Green light = sufficient (peacekeeping mandate = legitimacy)
- **Risk:** Bas légalement, très haut opérationnellement (security state)

### Conformité & Fiscalité

- **Taxation:** Minimal (government weak, no enforcement)
- **Focus:** NOT revenue (this is humanitarian deployment)
- **Governance:** UN coordination body = de facto authority (not Congolese government alone)

## 4. Roadmap Produit pour la RDC

### MVP (Novembre 2026) — Reconstruction Module Pilot

**Scope:** 500-1,000 IDP land recovery cases (Kivu region, if security permits)

**Modules inclus:**

1. **ERC-721 Reconstruction Titles** — IDP land recovery
   - Pre-conflict property verification (IDMC database matching)
   - Current occupant status (legal vs. illegal occupation)
   - Blockchain record = proof of IDP claim
   - Multi-party stakeholders (IDP + occupant + comité + mediator)

2. **Dispute Resolution Engine** — Land conflict mediation
   - 3-party mediation (IDP vs. occupant vs. community mediator)
   - UN observer (MONUSCO or OCHA representative, optional on-chain)
   - Blockchain-recorded outcome (binding at community level)
   - Appeal mechanism (to UN-recognized arbitrator)

3. **UN Reporting Integration** — IDMC + UNOPS tracking
   - Automated case status reporting (returned IDPs, resolved disputes, etc.)
   - Dashboard for UN coordination (real-time case tracking)
   - Impact metrics (# IDPs assisted, # cases resolved, # conflicts prevented)

4. **Succession (Secondary, if IDP inheritance)**
   - Multi-system (Christian + customary Congolese + Islamic)
   - Rare in Year 1 (focus = land recovery, not succession)
   - Platform available for future use

## 7. Risques & Mitigations (VERY HIGH RISK)

### Risque #1 — SECURITY: Team Safety (TRÈS ÉLEVÉ)

**Description:** Kivu = active conflict zone; kidnapping, violence, instability.

**Impact:** Team injury/death, project abandonment, reputational damage.

**Mitigation:**
- **Zero on-ground presence in combat zones** (Kivu = no SafeLand staff)
- NGO partners = experienced in conflict (ACIDH = established, trusted, local)
- UN security liaison (real-time intelligence, threat assessment)
- Insurance (hostile environment coverage, kidnap & ransom if needed)
- Abort triggers: If Kivu escalates >10% month-over-month, project pauses immediately
- Executive decision gate: CEO must explicitly approve before T0 (security responsibility)

---

### Risque #2 — POLITIQUE: Government Instability (TRÈS ÉLEVÉ)

**Description:** RDC government weak, coup/transition risk, ministry champion leaves.

**Impact:** Ministry withdrawal, project stalls, government refuses to share data.

**Mitigation:**
- **UN backing = protection** (government respects peacekeeping mandate)
- Don't rely on government coordination (use NGO + UN instead)
- Embed in UN operational structure (not government ministry)
- Multi-year UN commitment (resilient to government change)

---

### Risque #3 — TECHNIQUE: Infrastructure Breakdown (TRÈS ÉLEVÉ)

**Description:** Power cuts (daily), bandwidth unreliable, servers seized/damaged.

**Impact:** Data loss, platform down for extended periods, operation disruption.

**Mitigation:**
- **Offline-first architecture** (critical for conflict zone)
  - Paralegal staff work offline, sync when connectivity available
  - Batch processing (not real-time)
  - Local data cache (no cloud-only dependency)
- Server redundancy (2 locations, one Kinshasa + one regional if possible)
- Satellite internet backup (if budget allows)
- Paper backup (critical IDP records printed, stored secure)

---

### Risque #4 — HUMANITARIAN: Scope Creep (Moyen-Haut)

**Description:** UN/NGOs ask for additional modules (health data, gender-based violence tracking, etc.).

**Impact:** Scope explosion, timeline slip, budget overrun.

**Mitigation:**
- **Scope freeze:** Land recovery ONLY in Year 1 (no health/GBV/education)
- Phased approach (Year 1 land, Year 2+ other sectors if requested)
- Contracts specify scope clearly (SAU signed, scope change = formal variation)

---

### Risque #5 — RÉPUTATIONAL: Association avec Zone Conflit (Moyen)

**Description:** SafeLand associated with Kivu violence (media framing risk).

**Impact:** Reputational damage, investor concerns, team morale.

**Mitigation:**
- **UN partnership = legitimacy** (not a private company acting alone)
- Messaging: "humanitarian reconstruction" (not "business expansion")
- Press strategy: Controlled announcements via UN (not SafeLand directly)
- Impact storytelling: Document IDP returns (powerful narrative)
